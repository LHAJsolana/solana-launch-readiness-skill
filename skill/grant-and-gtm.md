# Founder GTM And Grant Materials

Use this file for launch communication, founder docs, grant submissions, demo readiness, user onboarding, and support content.

## Founder Launch Checklist

- Clear product name, one-line description, target user, and launch scope.
- Mainnet readiness report completed.
- Known risks and limitations documented.
- Official links ready: app, docs, X, Discord/Telegram, GitHub, explorer addresses.
- Support process and escalation path ready.
- Demo flow tested on fresh device and wallet.

## README Improvement Checklist

- What the project does.
- Why Solana is used.
- Architecture overview.
- Local setup.
- Environment variables with placeholders.
- Devnet/localnet instructions.
- Mainnet deployment notes.
- Security boundaries and no-secrets warning.
- Known limitations.
- License.

## Superteam/Colosseum Style Submission Helper

Answer concisely:

- Problem: who has the pain and why now?
- Solution: what the product does in one paragraph.
- Solana fit: why Solana's speed, composability, wallets, payments, tokens, or ecosystem matter.
- Traction: users, pilots, demos, metrics, community, integrations.
- Technical progress: repo, deployed program IDs, demos, tests, roadmap.
- Launch readiness: security, monitoring, incident plan, support.
- Ask: grant, intros, users, audit, infra credits, ecosystem support.

Do not invent traction, partnerships, users, TVL, revenue, or endorsements.

## X/Twitter Launch Thread Template

```markdown
1/ We are launching [project], a Solana [category] for [target user].

2/ The problem: [specific pain].

3/ What it does: [core workflow].

4/ Why Solana: [speed/cost/composability/wallets/payments].

5/ What is live today: [mainnet/devnet/demo scope].

6/ Safety and transparency: [authorities, audits if real, limitations, risk notes].

7/ Try it: [official link]. Support: [support link].
```

## Demo Video Checklist

- Start with the user problem.
- Show wallet connection safely.
- Show one successful core action.
- Show confirmation and explorer link.
- Keep private keys and admin tools out of frame.
- End with status, roadmap, and official links.

## Product Hunt-Style Copy

- Tagline: one sentence, no jargon.
- Description: problem, workflow, outcome.
- Maker comment: why you built it and what feedback you want.
- First comment: official links and support.

Use only if relevant to the launch channel.

## Community Onboarding

- First 5-minute guide.
- Wallet safety warning.
- Network and SOL fee explanation.
- FAQ for failed transactions.
- Official links and anti-phishing note.
- Support expectations and response time.

## Support FAQ

- Which wallets are supported?
- Which network is this on?
- Why did my transaction fail?
- I signed but nothing happened. What now?
- Where can I find the transaction signature?
- How do I avoid scam links?
- What authorities does the team retain?
- Where are docs and status updates?

## Responsible Claims

Avoid:

- Fake partnerships.
- Fake TVL, users, revenue, or volume.
- Guaranteed returns.
- Price predictions.
- Misleading performance claims.
- Undisclosed paid promotion.
- Claims of audit completion unless an actual audit report exists.
